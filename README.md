# 災害時医療・インフラ予測マップ

神奈川県を対象に、**人口密度・避難所・医療機関・公衆トイレ・地価**を1枚の地図に重ね、
生成AIに自然言語で相談できる防災マップです。

> 災害が起きたとき、自分の家の近くの避難所は混むのか。
> 持病のある家族が頼れる病院はどこか。水道が止まったら、近くに使えるトイレはあるか。
> 家を探すとき、災害に強くて地価の手頃なエリアはどこか。
>
> 「自分と家族が助かるために知りたいこと」を1枚の地図で確認できるようにしました。

---

## できること

| 機能 | 内容 |
|---|---|
| 人口密度メッシュ | 国勢調査の地域メッシュを 1km / 500m / 250m で切り替え表示。避難所の**混雑予測**に使う |
| 避難所 | 「命を守る（指定緊急避難場所）」と「生活する（指定避難所）」を区別。対応災害種別で絞り込み |
| 医療機関 | 診療科で絞り込み。持病を持つ家族の避難先を検討できる |
| 公衆トイレ | 多機能トイレ（車椅子・オストメイト対応）のみに絞り込み |
| 地価 | 価格帯で色分けした正方形アイコン。上限スライダーで予算内のエリアだけ表示 |
| AI防災アシスタント | 「一番近い避難所は？」等に自然言語で回答。**回答に登場した施設を地図上で発光させる** |
| 基準地点 | ブラウザの現在地 / 住所検索 / 神奈川県庁（既定）から選択 |

## データ

すべて公的なオープンデータを取り込み、クレンジングして使用しています。

| データ | 件数 | 出典 |
|---|---:|---|
| 避難所 | 4,265 | 国土地理院 指定緊急避難場所・指定避難所データ |
| 医療機関 | 310 | 厚生労働省 医療機関届出情報 |
| 公衆トイレ | 1,953 | 神奈川県オープンデータ |
| 地価公示 | 1,787 | 国土交通省 地価公示 |
| 人口メッシュ | 1,852 / 6,250 / 20,896 | 総務省 国勢調査 地域メッシュ統計 |

## 技術スタック

- **バックエンド**: Python 3.10+ / Azure Functions（V2 プログラミングモデル）
- **フロントエンド**: Vanilla JS（ESM、ビルド不要）/ Azure Maps Web SDK v3
- **テスト**: pytest 332件 / Vitest 215件（計 547件）
- **LLM**: OpenAI Chat Completions API

### アーキテクチャ

```
ブラウザ ──> Azure Static Web Apps（静的配信）
                    │ CORS
                    ▼
             Azure Functions（Python）
                    ├─ data/processed/*.json（クレンジング済みデータ）
                    ├─ 国土地理院 住所検索API
                    └─ OpenAI API
```

APIキーはすべて環境変数から読み込み、**ソースコードには一切含めていません**。
Azure Maps のキーはフロントに直書きせず `/api/config` 経由で配信しています。

## 設計上の判断

実装を進める中で方針を変えた箇所と、その理由を記録しています。

### 1. ジオコーディングに Azure Maps ではなく国土地理院APIを採用

地図描画と同じ Azure Maps で住所検索も賄うのが自然に見えましたが、実測すると日本の住所をほとんど解決できませんでした。

| 住所 | Azure Maps | 国土地理院 |
|---|---|---|
| 市街地の丁目・番地 | `search/address` 0件 / `search/fuzzy` 9〜15km ずれ | 解決可 |
| 横浜市中区日本大通1（県庁） | 0件 / 9.8km ずれ | **0.01km** |

9km ずれた基準地点から「最寄りの避難所」を案内するのは、防災アプリとして許容できません。
キー不要で日本の住所に特化した国土地理院APIへ切り替えました。
[`src/services/geocoding.py`](src/services/geocoding.py) で `Geocoder` を抽象化しているため、環境変数で差し替え可能です。

### 2. 地価×防災施設の集計をビルド時に移動

「地価が安く避難所が近いエリア」をAIに答えさせるには、町丁目ごとの集計が必要でした。
当初リクエスト時に計算していましたが、1,787地点 × 4,265避難所の距離計算で **`/api/landprice` が51.43秒** かかりました。

[`scripts/clean_landprice.py`](scripts/clean_landprice.py) でビルド時に集計する方式に変更し、**0.18秒**になっています。

### 3. 色分けを線形から分位点ベースへ

地価は 1,170円〜15,600,000円と4桁の幅があり、最小値〜最大値を線形に分割すると **99.8%が最下位の色帯に入り**、ほぼ単色の地図になりました。
分位点（5/30/55/80/95%）で区切ることで、各帯が 24.7 / 25.2 / 25.1 / 15.0 / 5.0% に分散しています。

### 4. AIが見つけた施設の照合を「名前」から「距離」へ

AIの回答文に出てきた施設を地図上で強調する機能で、トイレだけ一度も反応しませんでした。
調べると **1,953件中1,875件（96%）が「公衆トイレ」という同じ名前**で、AIが名前を書かず「0.117km先」とだけ答えていたためです。
[`frontend/src/highlight.js`](frontend/src/highlight.js) で距離による照合を併用しています。

### 5. 避難所データは「ディレクトリ名を信用しない」

国土地理院の配布データは、指定緊急避難場所と指定避難所のディレクトリ名と中身が入れ違っていました。
ファイル名ではなく**列の顔ぶれ**から種別を判定しています（[`scripts/clean_shelters.py`](scripts/clean_shelters.py)）。

### 6. 公開にあたってのレート制限

`/api/chat` は OpenAI の従量課金を消費するため、認証なしで公開するとURLを知られただけで請求が青天井になります。
[`src/services/ratelimit.py`](src/services/ratelimit.py) で二段構えの制限をかけています。

- IP単位: 10回/時間（移動窓）
- 全体: 200回/日 ← IPを変えられても効く上限

カウンタはプロセス内メモリのため、スケールアウトすると実効上限が緩みます。
公開時は Function App のスケール上限を1に固定しています。本番規模では Redis 等の共有カウンタへ差し替える前提で `RateLimiter` を抽象化しています。

## 開発

### セットアップ

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt -r requirements-dev.txt
npm install
cp .env.example .env   # キーを記入する
```

### 起動

```bash
func start        # API      → http://localhost:7071
npm run serve     # 静的配信 → http://localhost:5173
```

`frontend/src/api.js` が静的配信ポートを検出し、自動的に `localhost:7071` のAPIを参照します。

### テスト

```bash
pytest      # バックエンド 332件
npm test    # フロントエンド 215件
```

新機能は実装コードより先にテストを書く方針です（`CLAUDE.md`）。

### データのクレンジング

`data/raw/` に配布データを置いて実行すると `data/processed/` に生成されます。

```bash
python scripts/clean_mesh.py        # KEY_CODE から緯度経度を逆算
python scripts/clean_shelters.py
python scripts/clean_hospitals.py
python scripts/clean_toilets.py
python scripts/clean_landprice.py   # 町丁目別の集計も生成
```

人口メッシュの `KEY_CODE`（8〜11桁）は JIS X 0410 の規格に沿って矩形座標を数学的に逆算しています。

## 免責

本アプリの情報は公開データに基づく参考情報です。
災害時は必ず自治体の公式発表を確認してください。AIの回答は地図上のデータのみに基づき、生成AIの性質上誤りを含む可能性があります。
